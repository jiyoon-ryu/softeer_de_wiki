# 2026-07-02 회고

## 1. 오늘의 리뷰

오늘은 DA에 대해 학습하면서 데이터가 단순히 저장되는 것이 아니라, 목적에 맞게 구조화되고 관리되어야 한다는 점을 배웠다. 데이터를 잘 다루기 위해서는 분석이나 활용 이전에 데이터의 구조, 흐름, 관계를 이해하는 것이 중요하다는 것을 알게 되었다.

또한 두 번째 미션으로 SQL 관련 개념을 학습하고, 팀원들과 각자 이해하기 어려웠던 키워드에 대해 정리했다. 나는 `TOP`, `ALL`, `ANY`, `LIKE`에서 사용하는 `%`와 `_` 와일드카드에 대해 정리했다. 처음에는 SQL 문법이 DBMS마다 거의 비슷할 것이라고 생각했지만, SQLite에서는 `TOP`, `ALL`, `ANY` 같은 문법이 그대로 지원되지 않는다는 점을 알게 되었다.

팀 위키를 작성하면서 팀원들이 헷갈렸던 개념도 함께 확인했다. `SELECT INTO`, `UNION`과 `UNION ALL`, `PROCEDURE`, `WHERE`와 `HAVING`, `NULL` 비교, `AND`와 `OR`의 우선순위 등 각자 어려워한 부분이 달랐고, 이를 공유하면서 SQL 문법을 더 넓게 이해할 수 있었다.

## 2. Keep

* 수업에서 배운 DA 개념을 단순히 이론으로만 받아들이지 않고, 실제 데이터 관리와 SQL 학습에 연결해서 이해하려고 한 점이 좋았다.
* SQL 문법을 사용할 때 단순히 “된다/안 된다”로 끝내지 않고, 왜 SQLite에서는 지원되지 않는지와 어떻게 대체할 수 있는지까지 확인한 점을 유지하고 싶다.
* 팀원들과 각자 어려웠던 키워드를 공유하면서 내가 놓친 개념까지 함께 학습할 수 있었다.
* 개인적으로 헷갈렸던 개념을 위키에 짧게라도 정리해 둔 점이 좋았다.

## 3. Problem

* 처음에는 SQL 문법이 모든 DBMS에서 거의 동일하게 동작할 것이라고 생각해서, SQLite에서 지원되지 않는 문법을 만났을 때 혼란스러웠다.
* `ALL`과 `ANY`처럼 서브쿼리와 함께 쓰이는 문법은 개념적으로 바로 와닿지 않았다.
* 팀 위키를 작성할 때 개념을 정리하는 데 집중하다 보니, 실제 예시 쿼리를 더 다양하게 작성해 보지는 못했다.
* 배운 내용을 말로 설명하려고 하면 아직 정확한 용어로 정리하는 데 시간이 걸렸다.

## 4. Try

* 다음에는 SQL 문법을 학습할 때 내가 사용하는 DBMS에서 지원되는 문법인지 먼저 확인해 보겠다.
* 헷갈리는 SQL 키워드는 개념 설명만 적는 것이 아니라, 예시 쿼리와 실행 결과까지 함께 정리해 보겠다.
* 팀 위키를 작성할 때 다른 팀원의 정리도 읽고, 내가 몰랐던 부분은 따로 개인 위키에 추가로 정리하겠다.
* DA 개념을 배울 때도 단순 암기보다는 “이 개념이 실제 데이터 관리나 파이프라인에서 왜 필요한가?”를 연결해서 생각해 보겠다.

## 5. 한 줄 회고

오늘은 DA 개념과 SQL 미션을 통해 데이터는 단순히 저장하는 것이 아니라, 사용하는 환경과 목적에 맞게 구조화하고 다루어야 한다는 점을 배웠다.
