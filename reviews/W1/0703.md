# 2026-07-03 회고

## 1. 오늘의 리뷰

오늘은 ETL 파이프라인 미션을 진행하면서 웹에서 GDP 데이터를 추출하고, 필요한 형태로 변환한 뒤 SQLite 데이터베이스에 저장하는 과정을 학습했다.

BeautifulSoup과 requests를 사용해 웹페이지에서 데이터를 가져오는 방법을 실습했고, pandas를 이용해 필요한 컬럼을 정리하는 작업을 했다. 또한 SQLite에 데이터를 저장하면서 `CREATE TABLE`, `INSERT`, 데이터 조회 흐름을 확인했다.

추가로 GDP 데이터를 Wikipedia가 아닌 IMF 같은 공식 데이터 출처에서 가져오는 방법, 데이터가 갱신될 때 과거 데이터를 어떻게 관리해야 하는지에 대해서도 고민했다.

## 2. Keep

* 코드를 직접 실행하면서 ETL의 Extract, Transform, Load 과정을 단계별로 이해하려고 한 점이 좋았다.
* 단순히 코드를 작성하는 데서 끝내지 않고, 데이터 출처의 신뢰성과 갱신 문제까지 생각해 본 점이 좋았다.
* 모르는 라이브러리나 개념이 나오면 바로 찾아보고 적용해 보려는 태도를 유지하고 싶다.

## 3. Problem

* 처음에는 데이터를 가져오는 것에만 집중해서, 데이터가 나중에 갱신되었을 때 기존 데이터와 어떻게 구분할지까지는 깊게 생각하지 못했다.
* Wikipedia에서 가져온 데이터가 항상 정확하고 최신이라고 가정할 수 없다는 점을 뒤늦게 인식했다.
* 코드의 각 단계가 ETL 중 어떤 과정에 해당하는지 명확히 구분해서 설명하는 데 아직 익숙하지 않았다.
